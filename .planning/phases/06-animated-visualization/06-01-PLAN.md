    Create src/hooks/useAnimationSequence.ts — a React hook that drives the animation loop.

    **Types to define in the file:**

    ```typescript
    type AnimationPhase = 'idle' | 'playing' | 'complete';

    interface PieceAnimState {
      offsetX: number;  // translate offset from final position (starts at startX - finalX, ends at 0)
      offsetY: number;  // translate offset (starts at startY - finalY, ends at 0)
      opacity: number;  // starts at 0, ends at 1
    }

    interface UseAnimationSequenceReturn {
      phase: AnimationPhase;
      getPieceState: (pieceId: string, instanceIndex: number) => PieceAnimState;
      wasteOpacity: number;
      skipToEnd: () => void;
    }
    ```

    **Hook signature:**
    ```typescript
    export function useAnimationSequence(
      boards: BoardLayout[] | null,
      resultKey: number  // incremented each optimize click to retrigger
    ): UseAnimationSequenceReturn
    ```

    **Implementation details:**

    1. **State machine:** Use `useState<AnimationPhase>` — starts 'idle', transitions to 'playing' when boards changes from null to non-null (or resultKey changes), transitions to 'complete' when all pieces finish.

    2. **Animation data (useRef):** On trigger, compute all piece animation data once:
       - For each board, for each piece: call `computeStartPosition` to get start coords
       - Compute offset: `offsetX = startX - piece.x`, `offsetY = startY - piece.y`
       - Call `computeDelays` to get per-piece delay and duration
       - Store in a `Map<string, { offsetX: number; offsetY: number; delay: number; duration: number }>` ref

    3. **rAF loop (useEffect):** When phase is 'playing':
       - Record start timestamp on first frame
       - Each frame: compute elapsed time, for each piece determine progress `t = clamp((elapsed - delay) / duration, 0, 1)`, apply `easeOutCubic(t)`, compute current `offsetX = initialOffsetX * (1 - eased)`, `opacity = eased`
       - Store current values in a ref (NOT useState per frame — avoid React re-renders)
       - Use a single `useState` counter incremented each frame to trigger re-render (one state update per frame, React 19 batches this)
       - When all pieces have t >= 1, set phase to 'complete'

    4. **wasteOpacity:** After all pieces complete, fade waste in over 300ms. Track with same rAF loop. Return current waste opacity (0 during piece animation, 0->1 after pieces done).

    5. **getPieceState function:** Looks up piece key `${pieceId}-${instanceIndex}` in the current animation values ref. Returns `{ offsetX, offsetY, opacity }`. Returns `{ offsetX: 0, offsetY: 0, opacity: 1 }` if key not found or phase is 'complete'.

    6. **skipToEnd:** Sets phase to 'complete', cancels rAF. All subsequent getPieceState calls return final values. wasteOpacity returns 1.

    7. **Cleanup:** Cancel rAF in useEffect cleanup. Also cancel when resultKey changes (re-trigger).

    8. **Re-trigger:** When resultKey changes and boards is non-null, reset to 'playing' and restart animation from scratch.

    9. **Null boards:** When boards becomes null (input changed, result cleared), set phase to 'idle', cancel any running animation.

    **Performance notes per research:**
    - Use useRef for mutable animation state (offsets, timestamps)
    - Single useState frame counter for re-renders (not per-piece state)
    - React 19 batching handles the single setState in rAF callback
