    Modify 5 files to wire the animation hook into the visualization pipeline.

    **1. PieceRect.tsx — Add animation offset prop:**
    - Add optional `animOffset?: { offsetX: number; offsetY: number; opacity: number }` to PieceRectProps
    - Wrap the existing `<g>` content in an outer `<g>` with:
      - `transform={animOffset ? \`translate(${animOffset.offsetX}, ${animOffset.offsetY})\` : undefined}`
      - `opacity={animOffset ? animOffset.opacity : 1}`
    - Keep all existing rendering (rect, text, events) unchanged inside the inner `<g>`
    - When animOffset.opacity is 0, the piece is invisible (off-screen and transparent)

    **2. WasteRect.tsx — Add opacity prop:**
    - Add optional `opacity?: number` to WasteRectProps (default 1)
    - Apply opacity to the existing `<rect>` element: `opacity={opacity ?? 1}`
    - Keep fill pattern and stroke unchanged

    **3. BoardDiagram.tsx — Pass animation state to children:**
    - Add new props: `getPieceState?: (pieceId: string, instanceIndex: number) => { offsetX: number; offsetY: number; opacity: number }`, `wasteOpacity?: number`, `animPhase?: 'idle' | 'playing' | 'complete'`
    - In the pieces map, compute animOffset for each piece: `const animOffset = getPieceState?.(piece.pieceId, piece.instanceIndex)`
    - Pass `animOffset` to each `<PieceRect>`
    - Pass `opacity={wasteOpacity}` to each `<WasteRect>`
    - Suppress tooltips during animation: only set tooltip state if `animPhase !== 'playing'`. Modify handlePieceEnter to early-return if animPhase is 'playing'.

    **4. CuttingDiagramList.tsx — Hook up useAnimationSequence:**
    - Add `resultKey: number` to CuttingDiagramListProps
    - Import and call `useAnimationSequence(result.boards, resultKey)` to get `{ phase, getPieceState, wasteOpacity, skipToEnd }`
    - Pass `getPieceState`, `wasteOpacity`, and `animPhase={phase}` to each `<BoardDiagram>`
    - Add click-to-skip: wrap the board list `<div>` with an `onClick` handler that calls `skipToEnd()` when `phase === 'playing'`
    - Set `cursor-pointer` on the wrapper div when `phase === 'playing'` to indicate clickability

    **5. page.tsx — Add resultKey and pass to CuttingDiagramList:**
    - Add `const [resultKey, setResultKey] = useState(0)` state
    - In `handleOptimize`, after `setOptimizationResult(result)`, call `setResultKey(prev => prev + 1)` to trigger animation
    - Pass `resultKey={resultKey}` to `<CuttingDiagramList>`
    - The existing input handlers already call `setOptimizationResult(null)` which will cause useAnimationSequence to receive null boards and reset to 'idle' — no additional changes needed for "input clears animation"
