    Modify 5 files to wire the animation hook into the visualization pipeline using CSS transitions.

    **1. PieceRect.tsx -- Add CSS transition animation props (research Pattern 1):**

    Add three new optional props to PieceRectProps:
    - `animated?: boolean` -- false = at offset position, true = at final position
    - `slideFrom?: { dx: number; dy: number }` -- offset from final position in SVG user units
    - `skipMode?: boolean` -- true = disable CSS transitions for instant jump

    Wrap the existing `<g>` content in an outer `<g>` with inline style:
    ```typescript
    const hasAnimation = animated !== undefined && slideFrom !== undefined;
    const translateX = hasAnimation ? (animated ? 0 : slideFrom.dx) : 0;
    const translateY = hasAnimation ? (animated ? 0 : slideFrom.dy) : 0;
    const opacity = hasAnimation ? (animated ? 1 : 0) : 1;

    <g style={{
      transform: `translate(${translateX}px, ${translateY}px)`,
      opacity,
      transition: skipMode ? 'none' : 'transform 400ms ease-out, opacity 300ms ease-out',
      willChange: hasAnimation && !animated ? 'transform, opacity' : 'auto',
    }}>
      {/* existing <rect>, <text> content unchanged */}
    </g>
    ```

    Key details:
    - CSS `translate(Xpx, Ypx)` inside SVG uses SVG user units (mm in this app) per research Pitfall 1
    - `willChange` only set during pending animation, reset to 'auto' after (research Pitfall 4)
    - `transition: 'none'` when skipMode=true for instant jump (research Pattern 5)
    - When animated/slideFrom are undefined (no animation context), render exactly as before -- backward compatible

    **IMPORTANT anti-pattern from research:** Do NOT animate SVG x/y attributes. Do NOT conditionally mount/unmount pieces (causes layout shift per Pitfall 3). Always render all pieces; control with transform+opacity.
