1. Replace `src/app/globals.css` with the Tailwind v4 CSS-first theme configuration using `@theme inline` and CSS custom properties. This approach generates utility classes like `bg-surface`, `text-accent`, `border-border` that auto-switch between themes. The file must contain:
   ```css
   @import "tailwindcss";

   /* Enable class-based dark mode for next-themes */
   @custom-variant dark (&:where(.dark, .dark *));

   /* Semantic color tokens -- @theme inline generates utility classes AND CSS vars */
   @theme inline {
     --color-surface: var(--surface);
     --color-surface-alt: var(--surface-alt);
     --color-text-primary: var(--text-primary);
     --color-text-secondary: var(--text-secondary);
     --color-accent: var(--accent);
     --color-accent-hover: var(--accent-hover);
     --color-border: var(--border);
   }

   /* Light theme values */
   :root {
     --surface: #ffffff;
     --surface-alt: #f8fafc;
     --text-primary: #0f172a;
     --text-secondary: #64748b;
     --accent: #d97706;
     --accent-hover: #b45309;
     --border: #e2e8f0;
   }

   /* Dark theme values (activated by .dark class from next-themes) */
   .dark {
     --surface: #0f172a;
     --surface-alt: #1e293b;
     --text-primary: #f1f5f9;
     --text-secondary: #94a3b8;
     --accent: #f59e0b;
     --accent-hover: #fbbf24;
     --border: #334155;
   }

   body {
     background-color: var(--surface);
     color: var(--text-primary);
     font-family: 'Inter', ui-sans-serif, system-ui, -apple-system, sans-serif;
   }
   ```
   Why `@theme inline` not `@theme`: The `inline` modifier prevents Tailwind from adding these as default theme values, keeping them purely referential to the CSS vars. This is the recommended approach from the Tailwind v4 docs for tokens that change dynamically (like theme switching).

   Color choices per CONTEXT.md locked decision: slate/zinc neutrals with amber accent for warmth. Light values: surface=#ffffff, surface-alt=#f8fafc (slate-50), text=#0f172a (slate-900), accent=#d97706 (amber-600). Dark values: surface=#0f172a (slate-900), surface-alt=#1e293b (slate-800), text=#f1f5f9 (slate-100), accent=#f59e0b (amber-500).

2. Update `src/app/layout.tsx` to wrap the app in ThemeProvider and UnitProvider:
   ```tsx
   import type { Metadata } from 'next';
   import { ThemeProvider } from 'next-themes';
   import { UnitProvider } from '@/contexts/UnitContext';
   import './globals.css';

   export const metadata: Metadata = {
     title: 'CutList Pro',
     description: 'Woodworking cut list optimizer',
   };

   export default function RootLayout({ children }: { children: React.ReactNode }) {
     return (
       <html lang="en" suppressHydrationWarning>
         <body>
           <ThemeProvider attribute="class" defaultTheme="dark" enableSystem={false}>
             <UnitProvider>
               {children}
             </UnitProvider>
           </ThemeProvider>
         </body>
       </html>
     );
   }
   ```
   Key details: `suppressHydrationWarning` on `<html>` prevents React warning from next-themes script injection. `defaultTheme="dark"` per DES-01. `enableSystem={false}` so dark is always the default. `attribute="class"` so .dark class triggers CSS variable swap.

   IMPORTANT: next-themes ThemeProvider is a client component internally but can be imported directly in server layout. If this causes a build error about "use client", create a thin wrapper `src/components/ThemeProvider.tsx` with `'use client'` that re-exports it.

3. Create `src/components/ui/ThemeToggle.tsx` as a client component. Full implementation:
   ```tsx
   'use client';

   import { useTheme } from 'next-themes';
   import { useEffect, useState } from 'react';

   export function ThemeToggle() {
     const { theme, setTheme } = useTheme();
     const [mounted, setMounted] = useState(false);

     useEffect(() => setMounted(true), []);

     if (!mounted) {
       return <div className="w-9 h-9" />;
     }

     return (
       <button
         onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}
         className="w-9 h-9 flex items-center justify-center rounded-lg
                    text-text-secondary hover:text-text-primary
                    hover:bg-surface-alt transition-colors"
         aria-label={`Switch to ${theme === 'dark' ? 'light' : 'dark'} theme`}
       >
         {theme === 'dark' ? (
           <svg className="w-5 h-5" fill="none" viewBox="0 0 24 24" stroke="currentColor" strokeWidth={2}>
             <circle cx="12" cy="12" r="5" />
             <line x1="12" y1="1" x2="12" y2="3" />
             <line x1="12" y1="21" x2="12" y2="23" />
             <line x1="4.22" y1="4.22" x2="5.64" y2="5.64" />
             <line x1="18.36" y1="18.36" x2="19.78" y2="19.78" />
             <line x1="1" y1="12" x2="3" y2="12" />
             <line x1="21" y1="12" x2="23" y2="12" />
             <line x1="4.22" y1="19.78" x2="5.64" y2="18.36" />
             <line x1="18.36" y1="5.64" x2="19.78" y2="4.22" />
           </svg>
         ) : (
           <svg className="w-5 h-5" fill="none" viewBox="0 0 24 24" stroke="currentColor" strokeWidth={2}>
             <path strokeLinecap="round" strokeLinejoin="round" d="M21 12.79A9 9 0 1 1 11.21 3 7 7 0 0 0 21 12.79z" />
           </svg>
         )}
       </button>
     );
   }
   ```
   The `mounted` state guard prevents hydration mismatch -- renders a same-size placeholder div until client hydrates. Sun icon shown in dark mode (click to go light), moon shown in light mode (click to go dark). Uses semantic color tokens so button adapts to theme automatically.
